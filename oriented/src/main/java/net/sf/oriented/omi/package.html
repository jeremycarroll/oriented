This package is the public interface, with {@link net.sf.oriented.omi.OM} as the key interface defining oriented matroids.

Use {@link net.sf.oriented.omi.Options} and {@link net.sf.oriented.omi.FactoryFactory} to get started. There is support for: mathematical sets {@link net.sf.oriented.omi.SetOf}, {@link net.sf.oriented.omi.UnsignedSet}; signed sets {@link net.sf.oriented.omi.SignedSet}, and sets of such sets {@link net.sf.oriented.omi.SetOfSignedSet}, {@link net.sf.oriented.omi.SetOfUnsignedSet}, {@link net.sf.oriented.omi.SetOf#powerSet()},{@link net.sf.oriented.omi.SetOf#subsetsOfSize(int)}.

Oriented Matroids can be viewed as circuits, vectors or maximum vectors via the {@link net.sf.oriented.omi.OMasSignedSet} interface, or as chirotopes via the {@link net.sf.oriented.omi.OMasChirotope} interface. The {@link net.sf.oriented.omi.OM#dual()} method gives the cocircuits, covectors and topes. Oriented Matroids can be entered as matrixes, in which ase the {@link net.sf.oriented.omi.OMasRealized} interface applies.
